Codex Agent in Docker

Overview

For each bug, a Codex agent runs inside an OSS-Fuzz container at the target commit. It reads AGENTS.md for shared knowledge, then iterates: test → diagnose → patch → build → test. After triggering, a post-agent verification confirms the crash, then a second agent minimizes the diff.

Success

Minimized bug_transplant.diff + testcase

Impossible

bug_transplant.impossible + reason

Always

Update AGENTS.md for next bug

Test / Diagnose / Patch
script/prompts/bug_transplant.md
Agent Loop

Transplant Methodology

1
Test the PoC on target commit. Build with sudo -E compile, run testcase. Same crash = done. Different crash or no crash = keep going.
2
If testcase is rejected early, input format changed. Patch the testcase binary with xxd/dd/printf rather than reverting format definitions.
3
Find what differs in the crash path. Read fix hint diff if available. git diff <buggy> <target> -- <file> on crash stack functions.
4
The blocker may not be a single "fix". Could be incidental validation, format change, or refactoring. Focus on what concretely prevents the crash.
5
Verify both directions. Crash WITH changes, no crash WITHOUT (git stash round-trip). Same sanitizer class, same code area, overlapping call chain.
↺ build & test after every change

Verify + Minimize
script/prompts/minimize_patch.md
Post-Agent Verification

Official Build Check

Force sudo -E compile (agent may have built non-standard). Run fuzzer, compare crash stack with original to confirm same vulnerability — not a new bug.

Separate Agent Session

Patch Minimization

Try reverting each file → if bug still triggers, that file was unnecessary. Then minimize within required files hunk by hunk. Confirm crash stable (3 runs).

Offline Dispatch Merge
bug_transplant_merge_offline.py
Overview

Two-Phase Approach

Each bug with a code diff gets a unique dispatch bit pre-assigned. Phase 1 wraps each patch independently on clean source. Phase 2 merges all wrapped diffs together. Since each bug uses a different bit, they don't interfere at runtime.

Dispatch Mechanism

The harness reads __bug_dispatch[] from the first byte(s) of each testcase. Each PoC gets its bit prepended. Local/testcase-only bugs get 0x00.

Runtime code
if (__bug_dispatch[B] & (1<<N)) — both old and new code preserved
Macros
Ternary: (dispatch ? new : old)
Struct/type
Applied directly (wider type is backward-compatible)
Example

  --bug_info $BUGINFO_PATH --build_csv ~/log/c-blosc2_builds.csv \
  --target c-blosc2 --keep-container

Phase 1: Wrap Each Bug
script/prompts/dispatch_wrap_offline.md
Per-Bug Wrapping

Dispatch-Wrap on Clean Source

1
Reset source to harness baseline (clean + harness modification only).
2
Agent reads the standalone patch and rewrites every hunk with dispatch gating.
3
Build with sudo -E compile to verify. Save wrapped_<bug_id>.diff.
4
Reset source again for the next bug. Each wrap is independent.
↺ retry up to N times per bug if wrap fails
Dispatch wrapping pattern
// Every runtime code change gets wrapped like this:
#include "__bug_dispatch.h"

if (__bug_dispatch[B] & (1 << N)) {
    // Bug's version (NEW — from the patch)
} else {
    // Original version (OLD — before the patch)
}

// Macros use ternary:
#define LIMIT ((__bug_dispatch[0] & (1 << N)) ? 4096 : 4080)

Phase 2: Merge All
script/prompts/merge_wrapped_patches.md
Code Agent

Compose Wrapped Diffs

All wrapped diffs are copied into the container. A code agent applies them one by one, resolving textual conflicts (duplicate #include, adjacent hunks from different bugs). Since dispatch bits are unique, no semantic conflicts.

1
Agent applies patches in chunks (max 15 per agent call).
2
Build after each chunk to localize failures.
3
Final verification: all testcases replayed with dispatch bytes.
