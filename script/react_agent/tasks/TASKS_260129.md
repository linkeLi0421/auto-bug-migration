# TASKS 2026-01-29: Remove check_build and run_fuzzer from OSS-Fuzz agent

## Problem

The `ossfuzz_apply_patch_and_test` tool runs three steps:
1. `compile` (build) - Required
2. `check_build` - Fails due to missing `test_all` executable (infrastructure issue)
3. `run_fuzzer` - Not needed for build verification

The `check_build` step causes false failures when the Docker container doesn't have the `test_all` executable, even though the actual compilation succeeded.

**Note**: Build verification will be done in `revert_patch_test.py` instead.

- [x] Remove `run_fuzzer_seconds` parameter
- [x] Remove `check_build_cmd` construction and execution
- [x] Remove `run_fuzzer_cmd` construction and execution

- [x] Remove `check_build_ok`, `run_fuzzer_ok` from return dict
