Conflicting types errors:
- Error pattern: "conflicting types for '__revert_*'" or "conflicting types for 'func'"
- This error occurs when a function declaration has a DIFFERENT signature than the function definition.

COMMON CAUSE:
- A previous fix for "implicit declaration of function" or "call to undeclared function" added a forward declaration with the WRONG return type or parameters.
- Example: The function definition is `void __revert_5f664a_foo(int x)` but a declaration `int __revert_5f664a_foo(int x);` was added.

FIX WORKFLOW:

1. IDENTIFY THE CONFLICT: The error shows both locations - the declaration and the definition.
   - Look at the error message to find which signature is expected vs. provided.
   - The function DEFINITION is the source of truth - declarations must match it.

2. FIND THE BAD DECLARATION: Use `list_patch_bundle` to find `_extra_*` hunks for the file.
   - Bad declarations are often in `_extra_<filename>` hunks added by make_extra_patch_override.
   - Use `get_patch(patch_key="_extra_<filename>")` to view the hunk contents.

3. FIX THE DECLARATION: Use `make_error_patch_override` on the `_extra_*` hunk to correct the declaration.
   - Match the return type EXACTLY to the function definition.
   - Match the parameters EXACTLY to the function definition.
   - If the declaration cannot be fixed, REMOVE it entirely from the `_extra_*` hunk.

CRITICAL RULES:
- Do NOT change the function DEFINITION to match a wrong declaration.
- Do NOT delete the function definition - that causes linker errors.
- The declaration must match the definition, not the other way around.
- If you added the wrong declaration, you must fix or remove it.

EXAMPLE:
Error: conflicting types for '__revert_5f664a_xmlRegisterCharEncodingHandler'
- Definition (line 1503): `void __revert_5f664a_xmlRegisterCharEncodingHandler(xmlCharEncodingHandlerPtr handler)`
- Declaration (in _extra_encoding.c): `int __revert_5f664a_xmlRegisterCharEncodingHandler(xmlCharEncodingHandlerPtr handler);`

Fix: Edit the `_extra_encoding.c` hunk to change `int` to `void`:
`void __revert_5f664a_xmlRegisterCharEncodingHandler(xmlCharEncodingHandlerPtr handler);`
