Patch-scope mode:
- Patch bundles are applied via `git apply --reverse`: in these diffs, `-` lines become additions.
- Build-log locations `/src/...:line` refer to migrated/patched code; patch tools (get_error_patch_context / make_error_patch_override / make_extra_patch_override) take that location.
- read_file_context reads from the raw V1/V2 source checkouts; only call it with KB-derived locations or get_error_patch_context pre_patch_* mapping (never with raw build-log line numbers).
- search_definition(version=...) only accepts `v1` or `v2` (never pass commit hashes like `f0fd1b`).
- For mapped-slice rewrites: read the BASE slice artifact (read_artifact) then call make_error_patch_override with minimal edits; do not drop unrelated lines or omit the tail.
- For undeclared symbol/type/macro issues that should be provided at file scope: prefer make_extra_patch_override to extend the file's `_extra_*` hunk. Do NOT "fix" by rewriting the function to remove the symbol usage.
- For incomplete-type errors (e.g. “incomplete definition of type …”, “sizeof to an incomplete type …”): prefer make_extra_patch_override to insert/upgrade the missing tag body definition in `_extra_*` (do NOT rewrite functions into no-ops to avoid field access/sizeof).
- Do NOT replace missing globals with local variables (that changes semantics); add the real declaration/definition via make_extra_patch_override.
- After generating any patch override (make_error_patch_override or make_extra_patch_override), you MUST run ossfuzz_apply_patch_and_test before returning final.
